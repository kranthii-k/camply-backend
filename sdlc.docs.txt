Generate complete SDLC documentation for this project.

Follow a structured Software Development Life Cycle (SDLC) format with these major sections:

# 1. Project Overview
- Give a clear summary of what this project is about.
- Explain its purpose, objectives, and real-world use case.
- Include a short problem statement and how this project solves it.

# 2. Feasibility Study
- Mention the feasibility in terms of technical, operational, and economic aspects.
- Describe why the chosen tech stack (React, Node.js, Express, MongoDB, etc.) is suitable for this project.

# 3. Requirements Specification
- Clearly separate **Functional Requirements** (features the system must have) and **Non-Functional Requirements** (performance, scalability, security, etc.).
- Include sample use cases or user stories if possible.

# 4. System Design
- Provide a **high-level architecture diagram** description (Frontend, Backend, Database, API interactions).
- Mention data flow between components.
- Describe key modules, classes, or major files and their responsibilities.
- Include brief explanation of database schema (collections/tables, fields, relations).

# 5. Implementation
- Describe how the system was developed step-by-step.
- Mention libraries, frameworks, and tools used.
- Include important configuration steps (server setup, environment variables, etc.).
- Outline folder structure and what each folder is used for.

# 6. Testing
- Explain the testing approach used (manual, unit testing, API testing, etc.).
- Mention sample test cases or how functionalities were verified.
- Describe how bugs were identified and resolved.

# 7. Deployment
- Describe how and where the project was deployed (e.g., Vercel for frontend, Render/Node server for backend, MongoDB Atlas for database).
- Include steps or commands to deploy.
- Explain how environment variables or configurations are handled securely.

# 8. Maintenance and Future Scope
- Describe how updates, bug fixes, or improvements can be managed.
- Mention planned or ongoing features (like authentication, role-based access, etc.).
- Suggest potential enhancements or scalability improvements.

# 9. Conclusion
- Summarize what was achieved in this project.
- Mention your learning outcomes and the impact of the project.

# 10. Author
- Include your name, role (e.g., "Developed by Kranthi K as part of an academic project"), and date.

Format the output as a **formal SDLC report in Markdown (.md)** with proper headings, bullet points, and indentation. Use clear professional language suitable for interview or documentation submission.
